1. 사용자 정보를 저장하기
- 뭔가를 저장하는 것을 상태(state)라고 한다
- HTTP는 stateless(무상태) : 서버는 사용자를 구별하지 못한다
  예를 들어 사진 5장을 포함한 111번글이 있다고 하자
  프론트는 111번글, 사진 5장 총 6개의 request를 서버로 보낸다
  서버는 이 6개의 요청을 보낸 프론트가 동일인인지 여부를 알 지 못한다
1-1. cookie : 사용자 정보를 사용자 컴퓨터에 저장 -> 보안에 매우 매우 취약
1-2. sessiion : 사용자 정보를 서버의 메모리에 저장
1-3. database : 사용자 정보를 영구 저장

1-1. Cookie
- 예를 들어 naver가 상태 정보를 사용자 pc에 저장
  쿠키에는 수명을 지정할 수 있다.(3600이라고 주면 1시간)
- naver에 접근할 경우 "네이버에서 수신한 쿠키"가 네이버로 자동 전송

1-2. HttpSession
- 사용자 상태를 서버에 저장 -> 사용자를 구별 못한다면서?
  sessionid라고 하는 쿠키를 이용해 사용자를 구별
- 시나리오
  1. A라는 사용자가 네이버에 접근(A가 가진 네이버 쿠키들이 네이버로 자동 전송)
  2. 네이버는 쿠키를 까서 sessionid를 찾는다
     sessionid가 없을 경우 새로운 sessionid를 생성해서 쿠키로 기록
     ★ session은 사용자가 session이 없으면 만들어진다 ★
  3. 이제 네이버는 요청의 sessionid가 같다면 동일한 사용자로 간주
